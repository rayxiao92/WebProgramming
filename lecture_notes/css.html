Cascading Style Sheets (CSS)

I urge you to look at my dear friend and former intern Chris Traganos' presentation on CSS: CSS_COMP20_2010.pdf

Why CSS?

  • HTML was never intended for formatting or styling purposes but to define the content of a document
  • Once you embed presentation inside of an HTML tag, it cannot be overridden
  • The bottom line: separate the structure and content
  • Gives the designer creative control of the content. What you can change:
    • Font and font sizes
    • Color
    • Layout
    • Position
    • Borders
  • Control the layout of content on multiple devices using multiple style sheets (and thus improving accessibility)
  • Provides visual feedback to users (Web Engineering)

Syntax

  • A CSS file contains a sequence of rules
  • Example rule: h1 { color: blue; }
    • h1 is known as the selector
    • color: blue; is known as a declaration
    • color is known as the property
    • blue is known as the value
  • The selector comes first, then the declarations are enclosed by brackets
  • A rule can have many declarations
  • A declaration is a property-value pair separated by a colon
  • Declarations always ends with a semicolon
  • You can put one declaration on each line
  • To make a comment in a CSS, use /* COMMENT HERE */

id selector vs class selector

  • id selector
    • Used to specify a style for a single, unique element
    • Uses the id attribute of the HTML element. IMPORTANT: the ID cannot start with a number
    • Defined with a #
    • Example (CSS): #warning { color:red; font-size:48px; }
    • Example (HTML): <div id="warning">Don't eat the yellow snow!</div>
  • class selector
    • Used to specify a style for a group of elements
    • Class selector is most often used on several elements
    • The class selector uses the HTML class attribute, and is defined with a .
    • In HTML, the class attribute can contain multiple class names, separated by spaces
    • Example 1 (CSS): .yay { color: green; }
    • Example 1 (HTML): <p>In March, there is <span class="yay">SPRING BREAK!</span></p>
    • Example 2: p.center {text-align:center;} - All paragraph elements with class="center" will be center-aligned

Inserting and Using CSS

1. As external file; declare in <head> section (preferred)

<link rel="stylesheet" href="default_style.css" />

2. Internally in <head> section (meh!)

<style>
	...
	...
	...
</style>

3. Inline within an element (yuck!)

<p style="...">Whoa there!</p>

NOTE: The attribute-value pair type="text/css" for <link> and <style> is no longer required. See http://stackoverflow.com/questions/5409114/is-type-text-css-necessary-in-a-link-tag.

The "Cascading" in Cascading Style Sheets

  • What if there is more than one style specified for an HTML element?
  • The styles will roll-up or "cascade" into a bigger rule
  • The last declaration wins!
  • Priorities:
    • Browser default
    • External style sheet
    • Internal style sheet
    • Inline style

Advanced Compositions

  • Concatenation, separated .
    • Example: p.highlight.small { color: red; } - Applies to all paragraph elements whose class contains highlight and small
  • "Or", separated by ,
    • Example: h1, h2, h3 {...} - Applies to <h1>, <h2> or <h3> elements
  • Descendants, separated by space
    • Example: form p {...} - Applies to all paragraph tags inside form elements
  • Child of, separated by >
    • Example: form > p {...} - Applies to all paragraph elements that are direct children of form elements
